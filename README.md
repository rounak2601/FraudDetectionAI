## System Architecture

```
Transaction → Kafka → Feature Engineering (Redis) → ML Ensemble → SHAP + LLM → Analyst Dashboard
```

Every transaction flows through 6 stages:

| Stage | Component | Technology |
|-------|-----------|-----------|
| 1. Ingest | Kafka Topic | Apache Kafka + aiokafka |
| 2. Enrich | Velocity Features | Redis Sorted Sets (sliding windows) |
| 3. Graph | Fraud Ring Detection | Neo4j 2-hop subgraph + GraphSAGE GNN |
| 4. Score | ML Ensemble | XGBoost (ONNX) + Isolation Forest + Meta-Learner |
| 5. Explain | Explainability | SHAP TreeExplainer + Ollama LLM narrative |
| 6. Decide | Analyst Workflow | React Dashboard + Audit Trail |

## ML Models

### XGBoost (Primary Scorer)
- Trained on IEEE-CIS Fraud Detection dataset (Kaggle)
- Exported to ONNX format for ~3x faster inference than native XGBoost
- 29 engineered features including velocity, behavioral, and card features
- Handles class imbalance with `scale_pos_weight`

### Isolation Forest (Anomaly Detector)
- Trained on normal transactions only — no fraud labels required
- Catches zero-day fraud patterns that do not match historical patterns
- Score normalized to [0,1] range for ensemble combination

### GraphSAGE GNN (Fraud Ring Detector)
- Built with PyTorch Geometric
- 2-layer GraphSAGE with 64 hidden dimensions
- Reads 2-hop subgraphs from Neo4j per transaction
- Detects coordinated fraud rings invisible to tabular models

### Meta-Learner (Ensemble Combiner)
- Logistic regression combining all three model scores
- Weighted: XGBoost 50%, GNN 35%, Isolation Forest 15%
- Calibrated probability output for consistent thresholding

## Explainability

Every transaction scoring above 0.7 probability receives:

1. **SHAP Values** — per-feature attribution showing which features pushed the score up or down
2. **LLM Narrative** — 2-sentence plain-English explanation generated by local LLM via Ollama
3. **Triggered Rules** — deterministic rule engine (high amount, burner email, velocity spike)

---

## Case Management

Fraud analysts can:
- View all flagged transactions in the investigation queue
- Click into any transaction for full SHAP + AI explanation
- Add investigation notes
- Approve (legitimate) or Block (confirmed fraud)
- All decisions stored with SHA-256 hash-chained audit trail

## Quick Start

### Prerequisites
- Docker Desktop
- Python 3.11+
- Node.js 20+
- Ollama — https://ollama.ai

### 1. Clone and start infrastructure
```bash
git clone https://github.com/rounak2601/FraudDetectionAI.git
cd FraudDetectionAI
docker-compose up -d
```

### 2. Set up Python environment
```bash
python -m venv env

# Windows
env\Scripts\activate

# Mac/Linux
source env/bin/activate

pip install -r requirements.txt
```

### 3. Pull LLM model
```bash
ollama pull qwen2.5:1.5b
```

### 4. Start the backend
```bash
uvicorn backend.main:app --reload --port 8000
```

### 5. Start the frontend
```bash
cd frontend
npm install
npm start
```

### 6. Seed test data (optional)
```bash
python scripts/seed_data.py
```

Visit **http://localhost:3000** to see the dashboard.
